## 📖 Definición

> Es una plataforma de integración para gestionar el ciclo de vida de APIs con seguridad de nivel empresarial y con capacidades avanzadas de puertas de enlace (gateway). WSO2 permite diseñar, proteger, publicar y analizar APIs.

## Capacidades básicas

### Diseñar y crear API

WSO2 APIM puede crear y gestionar varios tipos de API. Entre sus capacidades se encuentran:
- Admite diversos tipos de APIs: REST, GraphQL, WebSocket, WebSub/WebHook, Server-Sent Events (SEE), SOAP-to-REST y API de IA.
- Herramientas para importar APIs: Compatible con OpenAPI/Swagger, AsyncAPI y GraphQL SDL.
- Crear productos APIs: Combina múltiples APIs en un producto unificado.
- Controlar versiones y revisiones: Control de versiones con implementaciones basadas en revisiones.

### Puertas de enlace universales y federadas

Permite implementar APIs en multiples tipos de gateways con una administración centralizada:
- **Universal Gateway**: Puertas de enlace de nivel empresarial con gestión, seguridad y escalabilidad.
- **Integración con Kubernetes Gateway**.
- **Puertas de enlace federadas**: Permite implementar APIs en AWS API Gateway, Azure API Gateway, Kong Gateway y Envoy Gateway.
- **Multiples entornos**: Producción, pruebas y entornos con gateways personalizados.
- **Politicas de puerta de enlace**: Transformaciones, mediaciones, manejadores personalizados y aplicación de politicas.
